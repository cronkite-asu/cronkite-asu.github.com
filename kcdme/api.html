KCDME API material

What is an API ?

APIs allow programs to talk to each other via defined interfaces.

What can I do with an API?

APIs on the web are generally used to either retrieve information from other applications on the web to use in your applications, or to perform actions in a web application without the user accessing the website itself. For example :

  • Find information about the weather
  • Find a precise location from an address
  • Find reviews of businesses in an area
  • Send a tweet
  • Receive Phone calls and interact with voice callers
  • Create custom maps
  • Receive and make payments
  • Spread your content via facebook
  • Find out when the next bus is due
  • Create charts easily

SOAP? REST? XML? JSON?

The terms "SOAP" and "REST" describe two popular methods for accessing APIs. For simple APIs, REST is significantly easier to work with. Where you have an option, you should choose to work with REST apis.

The terms "XML" and "JSON" describe two popular formats for transferring information between web apis. JSON is easier to work with than XML.

Authentication

Many APIs will provide you with an "API key", which is used to authenticate your application against their API. You will often need to include this in your request.

Also, sometimes you will need to authenticate your user against the API, too, particularly if you wish to perform actions on their behalf.

RESTful APIs and URL queries

A good way to start learning about simpler REST APIs is to simply play with the structure of URLs. Here are some examples.

http://search.twitter.com/search.json?q=journalism is a twitter search API query in JSON. The documentation for this API is here. In what ways can we adjust the URL in order to get different information ?

  • add geolocation to the search - &geocode=33.442368,-112.073762,10mi

http://chart.apis.google.com/chart?cht=p3&chd=t:60,40&chs=400x100&chl=Facebook|twitter is a google chart API query, which results in an image. The google chart API documentation is here. In what ways can we adjust the URL to get a different chart?

  • Add more data points cht=p3&chd=t:50,30,20&chs=400x100&chl=facebook|twitter|foursquare
  • Change the chart type cht=p or cht=p3 or cht=pc cht=lc
  • Change the colour scheme chco=FF0000,0000FF

Many APIs have "sandboxes" or "playgrounds" to make it easier for you to experiment with them. They're usually mentioned in the documentation. In the case of google charts, this page is set up to help you explore their API.
